# 产品向中文 README 设计

## Context

当前仓库 README 偏功能清单与开发者手册（版本号、PAT、缓存路径、`storageKey`、Sigstore 命令等靠前），对小团队客户不够有吸引力。参考 Nacos 文《一份可信来源，终结 Skill 管理混乱》的叙事与语气，将 README 改造成**产品落地页**，用中文向团队推销：痛点、路径、能力、下载与上手、安全承诺。

下载量与仓库曝光偏弱，README 首屏需要能让访客在 30 秒内理解价值并点到下载。

## Goal

- 主要读者：小团队 / 公司（统一维护 Skills，同步到多人、多项目、多 Agent）
- 语气：客户语言、主张式标题；少技术术语
- 结构：产品落地页（方案 B），不写成长文博客
- 默认入口：推销版**中文写进根目录 `README.md`**（GitHub 默认展示）
- 英文：改为简短导读并链到中文 README，或另存 `README.en.md`（实现时二选一，优先「`README.md` = 中文主力 + 顶部可链英文简版」）

## Non-Goals

- 不写成完整「Skill 治理平台」叙事（不宣称审核流水线、label、回滚等我们不具备的能力）
- 不在 README 保留 Cosign / Sigstore 验证命令（**完全删除**）
- 不在 README 保留开发者命令、迁移说明、agent-presets、维护者 Secrets 说明
- 不把「本机主库」误写成「团队可信仓库」

## Product Model（文案必须遵守）

两层来源 + 一层分发，措辞固定：

1. **团队 Skills 仓库（可信来源）**  
   自建 **GitLab** 或 **私有 Skill 中心**（文案统一用「私有 Skill 中心」，不再写 Skill Hub）。

2. **本机主库（分发中枢）**  
   用户本机目录。收拢团队 Skills 与个人本地 Skills，**用于分发**到各级目录与各 Agent，**不是**团队仓库本身。

3. **同步到 Agent / 项目**  
   安全链接安装到 Cursor、Claude Code、Codex 等目标；不覆盖目标内已有真实文件。

收束句可用：  
「GitLab / 私有 Skill 中心负责团队哪份算数；本机主库负责装到我的各个 Agent；同步负责真正用上且不踩坑。」

## README Structure

### 1. 首屏定位

- 标题：`Skills Sync Manager`
- 副标题：给团队的 Skills 一份可信来源——统一维护，安全同步到 Cursor、Claude Code、Codex 等多个 Agent，告别复制粘贴和版本混乱。
- CTA：下载安装（Releases）+ 页内锚点「5 分钟上手」
- 不出现：技术栈、changelog 式功能版本清单、内部字段名

### 2. 团队痛点

场景开场（文档规范 / 接口文档类 Skill 从个人到团队的故事），然后 4 条：

1. 版本对不齐  
2. 同步靠人肉  
3. 状态看不见  
4. 共享缺入口（内部 Skills 散落个人电脑、群文件、临时仓库）

收束：**问题不在 Agent 太多，而在 Skill 没有统一入口。**

### 3. 我们怎么解决

- **标题（已定）：** 终结复制粘贴：从团队仓库直达每个 Agent  
- 正文按「团队仓库 → 本机主库 → 同步到 Agent」三步写清（见 Product Model）  
- 不夸大成 Nacos 级 Registry 治理

### 4. 你能得到什么

客户语言短列表（无版本号前缀）：

- 接入团队仓库（自建 GitLab / 私有 Skill 中心）  
- 本机主库分发到多 Agent / 项目  
- 一键启用与卸载  
- 状态一目了然  
- 粘贴 GitHub 链接即装  
- 私有仓库访问密钥就地管理  
- 启动按需刷新来源  
- 用着安心（不扫全盘、不覆盖、二次确认）  

弱化或省略：对话框焦点、macOS 控件等实现细节；应用内升级最多一句。

### 5. 下载与安装

- 链接 GitHub Releases  
- Windows / macOS / Linux 安装包类型一句话说明  
- **删除**全部 Cosign / Sigstore 验证说明与命令  
- 可选极短人话：若系统提示不明来源，按系统指引允许打开即可（不展开签名技术）

### 6. 5 分钟上手

团队路径五步：

1. 设置本机主库  
2. 接入团队仓库（GitLab 或私有 Skill 中心）；私有 GitLab 需要时提示配置访问密钥（系统凭证库），不单开长节  
3. 安装 Skills 到本机主库  
4. 添加 Agent / 项目目标  
5. 勾选同步  

不写：`repo/` 分层、`storageKey`、`runtime-cache`、`linkStrategy` 等。

### 7. 用着安心

- 只操作指定目录，不扫全盘  
- 不覆盖目标已有真实文件/文件夹  
- 只卸本工具创建的链接  
- 无效 Skill 不安装  
- 删除主库 Skill 二次确认且不可恢复  

### 8. 文末

- 无开发者专节；需要者自行看源码或后续单独 CONTRIBUTING  
- 可选：相关链接（Releases、私有 Skill 中心服务端仓库若需）一行即可  
- 英文：简短导读 + 链到中文，或 `README.en.md`

## File Plan

| 文件 | 动作 |
|------|------|
| `README.md` | 重写为上述中文产品落地页（默认入口） |
| `README.zh.md` | 与 `README.md` 同步为同一中文内容，或改为「见 README.md」跳转，避免双份漂移（实现时优先：**两文件内容一致**，或 zh 仅跳转） |
| `README.en.md`（可选） | 英文短导读；若采用则在中文 README 顶部链过去 |

## Tone Reference

借鉴 Nacos 主张式标题与场景叙事，但能力边界诚实对应本产品：本机分发 + 团队仓库接入，而非完整审核/标签/回滚平台。

## Success Criteria

- 未接触过仓库的团队负责人能在首屏理解「解决什么、怎么用、点哪里下载」  
- 「本机主库」与「团队仓库」不会被读成同一概念  
- 无 Cosign 命令块、无开发者 API/迁移长文  
- 全文中文；「私有 Skill 中心」用语统一
